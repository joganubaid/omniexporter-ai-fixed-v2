KEY FINDINGS
================================================================================

1. ARCHITECTURE PATTERNS:
   ✓ Most platforms use REST APIs with JSON
   ✓ Gemini uses RPC-style batchexecute endpoint
   ✓ ChatGPT uses WebSocket for real-time messaging
   ✓ Perplexity uses Server-Sent Events (SSE)

2. COMMON FEATURES:
   ✓ Conversation/thread management across all platforms
   ✓ User authentication and session management
   ✓ Model/capability configuration
   ✓ File and image handling
   ✓ Rate limiting and quota management

3. DIFFERENTIATION:
   → ChatGPT: Advanced features (Gizmos, memory, research)
   → Claude: Integration focus (Gmail, Calendar, Drive sync)
   → DeepSeek: PoW Challenge for rate limiting
   → Grok: Workspace and collaboration features
   → Perplexity: Search-first with collections
   → Gemini: Heavy use of Google services

4. SECURITY OBSERVATIONS:
   ✓ Most use HTTPs
   ✓ ChatGPT uses sentinel endpoints for requirements
   ✓ DeepSeek includes Cloudflare Turnstile
   ✓ Token/auth headers in request metadata

5. INTEGRATION POINTS:
   ✓ OAuth connectors (Grok)
   ✓ Google services integration (Claude, Grok)
   ✓ MCP (Model Context Protocol) support (Claude)
   ✓ Third-party analytics and monitoring

METHODOLOGY
================================================================================

Extraction Process:
1. Loaded each HAR file as JSON
2. Iterated through log.entries
3. Extracted request URL and method
4. Filtered for API-relevant keywords
5. Removed CDN/asset URLs
6. De-duplicated and sorted results
7. Categorized by endpoint type

Filter Keywords Used:
  - 'api', 'rest', 'backend'
  - 'chat', 'conversation', 'thread'
  - 'message', 'completion', 'batchexecute'

Exclusions:
  - URLs containing '/cdn/assets'
  - Static asset files (.js, .css, .wasm, .svg, .png, .jpg)
  - Query parameters (only base URL retained)
